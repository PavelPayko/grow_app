## Why

Сейчас каталог компетенций жёстко привязан к команде при создании (`competency_catalogs.team_id`), и у команды может быть только один активный каталог без возможности сменить привязку. Это мешает переиспользовать каталоги между командами, централизованно вести матрицу компетенций и менять каталог команды без пересоздания структуры.

## What Changes

- Каталоги компетенций становятся самостоятельными сущностями без `team_id` и флага `is_active` per team
- Добавляется ручная привязка каталога к команде через `teams.catalog_id` с возможностью изменения
- API и UI каталога переводятся на управление каталогами независимо от команд (список каталогов, CRUD по выбранному каталогу)
- Привязка/смена каталога команды выносится в управление командами (или отдельный UI-блок)
- Клонирование каталога меняется с «из команды в команду» на «дублирование каталога» (новый независимый каталог)
- Миграция данных: существующие каталоги отвязываются от команд, командам проставляется `catalog_id` по текущей активной привязке
- **BREAKING**: удаляются эндпоинты `POST /api/teams/:teamId/catalogs` и клонирование по `source_team_id`/`target_team_id`; добавляются эндпоинты для глобальных каталогов и привязки к команде

## Capabilities

### New Capabilities

_(нет — изменения затрагивают существующие capability)_

### Modified Capabilities

- `competency-catalog`: каталог — независимая сущность; CRUD и клонирование работают с каталогами напрямую, а не через команду
- `teams`: команда хранит опциональную ссылку на каталог; админ может назначить или сменить каталог
- `competency-assessment`: валидация каталога при создании/активации цикла опирается на привязку команды, а не на `competency_catalogs.team_id`
- `competency-matrix-ui`: экран каталога и матрицы используют каталог команды через `teams.catalog_id`

## Impact

- **БД**: миграция `competency_catalogs` (убрать `team_id`, `is_active`), добавить `teams.catalog_id`; обновить seed
- **API**: `competencyCatalogService`, `competencyCatalogController`, маршруты; `teamService`, `teamController`; `assessmentCycleService.validateCatalogForTeam`
- **UI**: `competency-catalog` (toolbar, modals, hooks, API-клиент); управление командами (привязка каталога)
- **Типы**: `ICompetencyCatalog`, DTO клонирования и создания каталога
- **Тесты/скрипты**: `verify-competency-matrix.js`, seed-скрипты
